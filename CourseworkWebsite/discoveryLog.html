Reviewing Literature

Our next lecture was about the literature review itself and what we are expected to do when it come to creating a literature review as well as how to research effectively. Throughout the lecture the idea that this is a literature review and not a literary review was hammered into our minds to make sure we would be creating the correct piece of writing. We were told that a literature review consists of :

  • What has been done before
  • Major concepts, relationships & patterns
  • Strengths, weaknesses, conflicts and gaps in research
  • Approaches to a topic
  • The context of future work

We were also warned about common mistakes students make such as incorrect referencing and using too many quotes.

When it came to doing actual research we were given a place to start. We were shown that the best material to read would be papers in academic journals therefore, we should focus on journal articles. The university’s library was displayed as a great resource to use in order to find reliable journal articles and other materials. When looking at a source material we should consider how well of a source it is by checking if it meets these requirements:

  • Appears in a refereed journal
  • Refers to other good sources
  • May be written by a known/respected author

Within the lecture we were given the website, coggle. The site make it easy to create a mind map as well as plan the structure of the document.

Within our practical session we were tasked to create notes and start researching around the topic of risk mitigation. We were to start with search terms and then expand on those and then start taking notes on the materials found in order to find out the core themes around this topic. To find these search terms and materials I used both google scholar as well as the university library website. Separating Form and Content

In this style lecture I learnt that the markup language should only be used for text that helps convey meaning while a stylesheet is used for telling the browser how to display the different sections of the markup. Markup can be separated into two groups, procedural and descriptive. Procedural tells the markup what to do, how to look but doesn’t specify why to do it. On the other hand, descriptive markup tells us what the markup means and not how to look or what to do. We are expected to use descriptive markup in order to separate style and content. Also, it allows for a different presentation to be set for the same content without needing to rewrite the code again and it enables machine processing. We can style our HTML outside of the actual HTML through the use of CSS.

Within the practical session I learnt how to create my own stylesheet which can be done through creating a .css file and then linking the css file to the HTML file by using the command in HTML I also learnt about using class selectors and id selectors. Class selectors should be used for multiple elements that will be styled in a certain way while id selectors can be used to style a specific element in the HTML.

Addressing the Web

This lecture was about links and how to use them in HTML. I learnt that to navigate a website you use links in order to go to the next page. I learn the structure of a URL is the scheme, username,password, host, port, path, query and fragment. The scheme refers to the format in which the request should be sent such as, https:// The hostname refer to ip addresses using DNS. DNS is the global record of domains to ip addresses such as .com or .ac.uk Ports allow for network services to be run such as HTTPS on port 443. The username and password can be provided within a URL but it is discouraged as it is a security risk. Query strings are allow for variables to be passed onto the web page such as name=bob. Fragments allow for the screen to be scrolled to a certain area of the web page such as index.html#cake.

I learnt how to create a URL in HTML. It is done through the use of the "a" tag in the format "a href = "url">Link Text/a". Also, The URL can be either relative or absolute and URLs that do not contain a scheme are assumed to be relative URLs.

Within the practical I cemented my knowledge on links as we were tasked with creating absolute and relative links as well as targets and fragments. These links were placed in a HTML file and then uploaded to my VM where I was able to to traverse different pages and go to different websites.
